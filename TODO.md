# 開発タスクリスト

## Phase 1（直近）

- [ ] **半前金設定**
  - 受注時に前金（例: 50%）を請求し、納品後に残金を請求する分割請求フロー
  - 案件マスタに前金率/前金額列を追加
  - 前金請求書・残金請求書の2回PDF生成
  - 入金管理も2段階に対応

- [ ] **メール送信機能**
  - 生成した見積書・請求書PDFを`GmailApp.sendEmail`で顧客に直接送信
  - 送信テンプレート（件名・本文）の設定シート管理
  - 送信履歴ログ

- [ ] **添付ファイル自動経理フォルダ格納**
  - PDF生成時に指定の経理用Googleドライブフォルダへ自動コピー
  - 年月別サブフォルダ自動作成
  - 帳票種別（見積/発注/請求）ごとの整理

## Phase 2（業務効率化）

- [x] **顧客マスタシート** ← v4.0.0で実装
  - 顧客マスタを分離して再入力不要に
  - 顧客ID紐づけ、住所・担当者・メールアドレスの一元管理
  - メール送信の宛先管理にも活用

- [ ] **入金消込機能**
  - 請求書に対する入金を紐づけて消込
  - 部分入金対応、未入金アラートの強化

## Phase 3（運用強化）

- [ ] **見積有効期限アラート**
  - 見積提示後N日経過で未受注の案件をアラート表示
  - フォローアップ漏れ防止

- [ ] **通知機能（Slack/チャット連携）**
  - 入金遅延・ステータス変更時にSlack/Google Chat等へ自動通知

- [ ] **月次レポート自動生成**
  - 月末にダッシュボードのKPIをPDF/スプレッドシートでエクスポート

## Phase 4（スケール対応）

- [ ] **権限管理・マルチユーザー対応**
